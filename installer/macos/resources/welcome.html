DisplayXR MCP Tools

Installs displayxr-mcp, the stdio↔socket adapter for the Model Context Protocol, into /usr/local/bin/, and flips on the MCP capability flag the DisplayXR runtime (and shell, when present) reads at startup. Once installed, the runtime spawns a per-PID MCP server that AI agents and voice clients can attach to for runtime introspection.

The capability marker lands at /Library/Application Support/DisplayXR/Capabilities/MCP/; removing or rewriting Enabled to anything other than 1 disables MCP without uninstalling the adapter.

Soft prereq: the DisplayXR runtime is the producer of MCP endpoints — without it, the adapter has nothing to attach to. Installing this first is fine; the capability flips on as soon as the runtime is present. The DISPLAYXR_MCP env var continues to work as a dev / CI force-enable override.
